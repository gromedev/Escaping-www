Conditional Access Policy Reference

Conditional Access policies are the core of the Entra ID Zero Trust security model. This guide covers common configurations.

Require MFA for All Users

Create a policy targeting "All users" with the grant control "Require multifactor authentication". Exclude emergency access accounts.

Block Legacy Authentication

Target "All users" and "All cloud apps". Under conditions, select "Client apps" and check "Exchange ActiveSync clients" and "Other clients". Set grant to "Block access".

Require Compliant Devices

Target specific user groups and cloud apps. Grant control: "Require device to be marked as compliant". This requires Intune enrollment.

Location-Based Access

Define named locations (trusted IPs, countries). Create policies that require additional controls when accessing from untrusted locations.
